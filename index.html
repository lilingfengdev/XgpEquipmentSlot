                    简介

                    • ✔Mysql Sqlite Redis 支持！
                    • ✔简单的套装效果！
                    • ✔以 SX2&3 或 AP3 作为前置插件！
                    • ✔(兼容附属 ZF-Runes ...更多的话私聊我看看 万一实现了呢)
                    • ✔自定义配置的额外装备栏！
                    • ✔理论支持 全版本 但是前置并不是全版本~ 不支持的话... 私信我改一下
                    • ✔没了喵

                     

                    ⚠️注意事项！

                    • 不支持数据库之间的数据转换！ 请在开服前确认数据库类型。 况且你也可以通过在config中配置sqlite的位置来实现跨服?
                    • 请在看完文档与配置文件的注释后，再向作者提问！

                    权限节点

                    • XgpES.bypass

                      跳过禁用原版背包检测权限。

                    插件命令

                    • help

                      • /xes help
                      • 获得命令帮助信息

                    • nbt

                      • /xes hbt
                      • 获得手中物品的nbt/1.12.2及以下版本会输出在插件文件夹的nbt.txt中

                    • openInv

                      • /xes openInv <装备栏名称>
                      • 打开指定名称(name)的装备栏。
                      • /xes openInv <装备栏名称> <玩家名称>
                      • 查看指定玩家的装备栏,记得确认大小写! OP可以直接拿玩家的装备。 支持离线玩家！
                      • /xes openInv
                      • 打开默认装备栏， 在config.yml中配置

                    • reload

                      • /xes reload
                      • 重载插件~ 请不要使用plugman启用捏!

                    • tag

                      • /xes tag add [tag]
                      • 手持物品,向手中的物品添加tag,用于配置中的检验.
                      • /xes tag remove
                      • 手持物品,移除本插件添加的tag

                     

                    配置装备栏！

                    在插件文件夹中会生成一个default.yml 路径在/XgpEquipmentSlot/EquipmentSlot/default.yml

                    在此文件夹中所有yml都会尝试读取，其中name不能重复。

                    这个配置大致有如下必填的根节点:

                    name 这个装备栏的名称,在插件中作为识别名。

                    GUI 配置这个GUI的排版与标题 标题支持占位符。

                    items 设置GUI中的物品信息与装备槽配置。

                    对items的子节点配置：

                    nbt <测试中功能，也许不稳定> 直接填入nbt, 下面的material、damage、amount就可以不用填了！ 对应nbt可以通过命令获取。

                    material 物品材质 从以下链接获取相应的物品！ 记得大写！

                    1.12:https://docs.zoyn.top/bukkitapi/1.12.2/org/bukkit/Material.html ​ 1.13+:https://bukkit.windit.net/javadoc/org/bukkit/Material.html

                    damage 特殊meta值 比如灰色玻璃板是7 橙色是1 (可不填)

                    amount 显示数量 （默认为1）

                    displayName 物品显示名

                    lore 物品的lore 支持占位符

                    设置装备槽

                    添加一个equipmentSlot 节点，即可把它设置为装备槽。 前面的物品会设置为无装备时显示的物品，有装备后会自动替换为穿戴的装备。

                    装备判断

                    你至少要对equipmentSlot添加一个子节点！

                    type 填写此项则会检验物品是否拥有该tag,可使用命令/xes tag add helmet 添加nbtTag 物品会添加"XgpES:helmet"nbt标签，物品交易可能会受到影响!

                    lore 使用lore作为判断依据,设置此项则也会对lore进行判断,只要lore中包含这个字符串就会成功,即使是 "一头盔甲" 。这两种方法有一种成功则可以穿戴。
